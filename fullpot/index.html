05 · Acessos e permissões

O que a FullPot precisa liberar

Pra que os agentes operem de ponta a ponta, a FullPot precisa liberar os acessos abaixo. A infraestrutura permanece toda na conta da FullPot, conforme a seção 06.

5.1

API do CRM próprio da FullPot

A FullPot construiu o CRM internamente, então os agentes vão operar pela API desse sistema. O time de tecnologia precisa disponibilizar:

  • Endpoints GET de leitura: contatos e leads, pipeline com negócios e estágios, atividades e tarefas, e as gravações e transcrições de chamadas dos vendedores
  • Endpoints POST ou webhooks de escrita: criação e atualização de registros, disparo de email e disparo de SMS
  • Documentação da API. Na falta dela, uma sessão técnica com o desenvolvedor responsável pra mapear os endpoints disponíveis
  • Chaves de API e credenciais de acesso com permissão de leitura e escrita
  • Ambiente de testes ou sandbox do CRM, se existir, pra validar as integrações sem tocar em dados de produção
  • Caso a API não cubra tudo, acesso de leitura direto ao banco de dados, somente leitura, como alternativa pros agentes de análise
Após uma sessão técnica com o time de tecnologia da FullPot pra mapear os endpoints existentes, entregamos a lista final de integrações e o que precisa ser exposto.
