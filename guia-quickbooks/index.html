Conecte seu QuickBooks Online e automatize a contabilidade

Guia passo a passo para ligar o QBO no seu computador, testar a conexao, lancar despesa vinculada a projeto e puxar relatorios financeiros. Tudo rodando na sua maquina, com suas credenciais.

1

O que essa skill faz

Ela integra a sua empresa ao QuickBooks Online usando a Accounting API (v3), com scripts simples em Node. Voce ganha quatro capacidades praticas:

  • Conectar o QBO via OAuth2 e manter o acesso renovado sozinho.
  • Testar a conexao para confirmar que as credenciais estao certas.
  • Lancar despesa vinculada a um projeto (a despesa fica atribuida ao projeto certo dentro do QBO).
  • Puxar relatorios financeiros como DRE (P&L), Balanco, Fluxo de Caixa e outros.
Para quem serve: dono de empresa ou operador financeiro que ja usa QuickBooks Online e quer parar de digitar despesa por despesa na mao, alem de puxar relatorio na hora que precisar sem abrir o painel.
2

Pre requisitos

Antes de comecar, confira que voce tem tres coisas:

  • Conta no QuickBooks Online. Para comecar sem risco, use o sandbox (empresa de teste com dados ficticios) que a conta de desenvolvedor ja cria.
  • Plano Plus ou Advanced se voce for usar Projects (despesa por projeto). No Simple Start e no Essentials o recurso de Projects nao existe. O sandbox ja vem como Plus.
  • Node 18 ou superior instalado. A skill nao usa nenhuma dependencia extra, so o Node.
conferir a versao do node
node --version

Se aparecer algo como v18 ou maior, esta pronto. Se der erro de comando nao encontrado, instale o Node primeiro em nodejs.org.

3

Obter as 4 credenciais na Intuit

O objetivo desta etapa e sair com quatro valores: Client ID, Client Secret, Realm ID (Company ID) e Refresh Token. Leva cerca de 10 minutos. Comece sempre pelo sandbox.

Parte 1 · Criar conta de desenvolvedor

  1. Abra developer.intuit.com.
  2. Clique em Sign up (ou Sign in se ja tiver login Intuit ou QuickBooks).
  3. Confirme o e-mail se for pedido.

Parte 2 · Criar o app (gera Client ID e Client Secret)

  1. Logado, va em Dashboard.
  2. Clique em Create an app.
  3. Escolha a plataforma QuickBooks Online and Payments.
  4. De um nome qualquer (ex: Minha Integracao) e marque o escopo com.intuit.quickbooks.accounting. Crie.
  5. No app, abra a aba Keys & credentials.
  6. Selecione a secao Development (nao Production).
  7. Anote o Client ID e o Client Secret.

Parte 3 · O sandbox (empresa de teste) ja vem pronto

  1. A conta de dev ja cria um sandbox. Confira em Dashboard > Sandbox.
  2. Anote o Company ID mostrado ali: esse e o seu Realm ID.
  3. Para habilitar Projects, abra a empresa de teste e va em Settings (engrenagem) > Account and settings > Advanced e ative Projects.
  4. Crie 1 projeto de teste em Projects > New project, escolha um cliente existente e salve. Assim o script encontra um projeto real.

Parte 4 · Gerar o Refresh Token (OAuth Playground)

  1. Abra o OAuth 2.0 Playground.
  2. No topo, selecione o SEU app, o ambiente Sandbox e o escopo Accounting.
  3. Clique Get authorization code e autorize com a empresa sandbox.
  4. Clique Get tokens. A tela mostra o Access Token (dura 1 hora, o script renova sozinho), o Refresh Token (dura cerca de 100 dias, esse voce guarda) e o Realm ID.
  5. Anote o Refresh Token.
Trate como senha. O refresh token da acesso de leitura e escrita a sua contabilidade. Nunca compartilhe, nunca cole em chat ou e-mail. Se algum dia vazar, revogue o app na Intuit e gere um novo.
4

Instalar a skill e preencher as credenciais

Com as 4 credenciais em maos, entre na pasta da skill pelo terminal e crie o arquivo local a partir do template.

copiar o template de credenciais
cp scripts/credentials.example.json scripts/credentials.local.json

Abra o arquivo scripts/credentials.local.json e cole os 4 valores. Deixe o environment como "sandbox" por enquanto e salve.

scripts/credentials.local.json
{
  "clientId": "SEU_CLIENT_ID",
  "clientSecret": "SEU_CLIENT_SECRET",
  "realmId": "SEU_REALM_ID",
  "refreshToken": "SEU_REFRESH_TOKEN",
  "environment": "sandbox",
  "minorVersion": "73"
}
Suas credenciais ficam so na sua maquina. O arquivo credentials.local.json e bloqueado pelo .gitignore, entao ele nunca vai para o GitHub e nunca e enviado para ninguem. Os valores nao saem do seu computador.
5

Rodar os 3 comandos

Sempre rode de dentro da pasta da skill. Comece pelo teste de conexao.

1 · Testar a conexao (faca isso primeiro)

Renova o token, le os dados da empresa e lista clientes e contas. Termina com a mensagem CONEXAO OK se estiver tudo certo.

terminal
node scripts/test-connection.js

2 · Lancar despesa vinculada a um projeto

Descobre os projetos, cria a despesa (Purchase) apontando para o projeto e le de volta para confirmar que o vinculo foi gravado. Se voce ainda nao tiver um projeto real, ele cria um sub cliente proxy e avisa a limitacao.

terminal
node scripts/create-project-expense.js
node scripts/create-project-expense.js --project="PROJ-001" --amount=150.50
node scripts/create-project-expense.js --project-id=58 --amount=42

3 · Puxar relatorios financeiros

Puxa a DRE (P&L) por padrao, imprime um resumo e salva o JSON cru em scripts/out/. Voce pode pedir outros relatorios: ProfitAndLoss, BalanceSheet, CashFlow, CustomerBalance, AgedReceivables, AgedPayables, GeneralLedger e TrialBalance.

terminal
node scripts/pull-reports.js
node scripts/pull-reports.js --report=BalanceSheet --start=2026-01-01 --end=2026-12-31
6

O que funciona x o que nao funciona

Cada item abaixo foi rodado de verdade contra um sandbox real da Intuit. Isso aqui e honesto: melhor voce saber o limite antes de prometer algo.

FUNCIONA

  • Renovar o acesso via OAuth2 (refresh token).
  • Ler empresa, clientes e plano de contas.
  • Descobrir os projetos existentes.
  • Criar despesa (Purchase) via API.
  • Vincular a despesa a um projeto (o vinculo persiste na leitura de volta).
  • Marcar a despesa como faturavel.
  • Puxar relatorios contabeis (DRE, Balanco e outros).

NAO funciona

  • Criar um Project pela API. O projeto precisa existir na tela do QBO; pela API so da para criar um sub cliente proxy.
  • Usar Projects sem plano Plus ou Advanced.
  • Ler o extrato bancario (o feed bruto do banco). A API expoe entidades contabeis e relatorios, nao o extrato.
  • Confiar no filtro de projeto direto na consulta; a skill puxa os clientes e filtra o projeto no resultado.
Dupla prova. Nao diga que funcionou so pela resposta da API. Confirme nas duas frentes: (1) a saida do script imprime o VEREDITO como SIM e o vinculo bateu; (2) na tela do QBO, abra o projeto na aba de custos e veja a despesa listada.

Checklist de validacao

Marque cada item conforme for confirmando. So diga resolvido quando tudo estiver batido.

Conexao

Imprimiu access token obtido (OAuth ok).
Imprimiu o nome da empresa (realm correto).
Listou clientes e contas sem erro.
Terminou com CONEXAO OK.

Despesa por projeto

Imprimiu Purchase criada com Id (POST aceito).
VEREDITO imprimiu SIM, o vinculo ficou gravado.
O valor do vinculo bate com o Id do projeto usado.
O alvo usado e um projeto real, nao apenas o sub cliente proxy.
Na tela do QBO, a despesa aparece atribuida ao projeto.

Relatorios

Imprimiu o nome do relatorio, periodo e moeda.
Imprimiu as linhas do relatorio pedido.
Salvou o JSON cru em scripts/out/.

Deu tudo certo no sandbox? So depois disso passe para producao: pegue as chaves da secao Production (Client ID e Client Secret sao diferentes), gere um novo Refresh Token apontando para a sua empresa real e troque o environment para "production". Producao exige que o app passe pela revisao da Intuit.
